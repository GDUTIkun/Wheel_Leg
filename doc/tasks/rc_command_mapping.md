# 模块名称

`rc_command_mapping`

## 当前迭代目标

完成从 `iBUS` 原始通道到统一控制命令的映射设计，固定归一化、死区、反向和 failsafe 行为。

## 任务 checklist

- [x] 固定通道语义确认流程。
- [x] 固定“映射确认需用户现场配合拨动摇杆和开关”的执行要求。
- [x] 固定连续量通道的归一化和死区规则。
- [x] 固定开关量通道的离散化规则。
- [x] 固定 `/cmd_vel`、`/control_mode`、`/body_cmd` 的映射职责。
- [x] 固定 failsafe 行为。

## 每个任务的输入

- `/rc/channels_raw`
- 通道标定参数：`min / center / max`
- 通道预处理参数：`reverse / deadzone`
- 模式映射阈值

## 每个任务的输出

- 连续量控制命令
- 模式切换命令
- 机身附加控制命令
- 失联时安全命令

## 完成标准

- 控制器只订阅统一命令，不订阅原始通道。
- `/cmd_vel`、`/control_mode`、`/body_cmd` 语义明确。
- 失联时会清零运动命令并退回安全模式。

## 验证方法

- 由用户现场单独拨动每个摇杆或开关，确认映射链路正确。
- 验证中位死区、端点归一化和反向配置。
- 验证模式切换开关稳定。
- 验证接收机断链或超时后进入 failsafe。

## 当前状态

- `[v]` 已完成代码实现与实机验证。
- 已确认真实遥控器通道布局：`CH2` 右摇杆上下、`CH1` 右摇杆左右、`CH3` 左摇杆上下、`CH4` 左摇杆左右、`CH5` 开关、`CH6` 三段开关。
- 已验证 `CH2 -> /cmd_vel.linear.x`、`CH4 -> /cmd_vel.angular.z`、`CH5 -> disabled/stand`、`CH6 -> stand/velocity` 映射符合预期。
