# 模块名称

`rc_ibus_node`

## 当前迭代目标

完成树莓派侧 `iBUS` 输入节点设计，固定串口接收、协议解包、原始通道发布和状态诊断行为。

## 任务 checklist

- [x] 固定 `/dev/ttyAMA3` 串口配置和运行前提。
- [x] 固定“是否有信息接收到”的串口联通性前置测试。
- [x] 固定 `iBUS` 32-byte 帧解包规则。
- [x] 固定 checksum 校验与坏帧丢弃规则。
- [x] 固定 `/rc/channels_raw` 输出语义。
- [x] 固定 `/rc/status` 输出语义。
- [x] 固定节点参数项与默认值。

## 每个任务的输入

- 接收机：`FlySky FS-iA6B`
- 协议：`iBUS`
- 串口：`/dev/ttyAMA3`
- 串口参数：`115200 8N1`

## 每个任务的输出

- 串口初始化规范
- `iBUS` 解包规范
- 错误计数与状态发布规范
- 原始通道 ROS2 topic 语义

## 完成标准

- 节点职责边界明确，不让控制器直接感知 `iBUS`。
- 合法帧、坏帧、超时和串口异常的处理规则全部固定。
- 原始通道 topic 可直接用于树莓派侧通道确认和排障。

## 验证方法

- 启动节点并确认串口打开成功。
- 在协议解析前先验证串口是否收到持续字节流。
- 观察 `/rc/channels_raw` 是否持续发布。
- 验证坏帧不会进入原始通道有效输出。
- 验证 `/rc/status` 能反映在线状态和错误计数。

- 已在 Raspberry Pi 上确认 `/dev/ttyAMA3` 可稳定接收 `FlySky FS-iA6B` 的 `iBUS` 帧。
- 已验证 `/rc/channels_raw` 与 `/rc/status` 可用于现场排障和通道确认。
